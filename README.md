# ResQNet

ResQNet is a web application that lets users report and track disaster incidents on a live interactive map.  
It helps communities share verified, location-based updates during emergencies such as fires, floods, and accidents, improving awareness and local coordination.

---

## Problem Statement
In many disaster situations, accurate ground-level information is either delayed or unavailable, which affects emergency response and resource distribution.  
Hearthwatch addresses this gap by offering a platform where users can instantly post incidents, attach images, and view nearby alerts, enabling faster, community-driven responses.

---

## Features
- Dynamic Map Interface – Visualize reports in real time using location data.  
- Incident Reporting – Submit events with title, category, and optional image.  
- Categorization System – Filter by main category or subtype for clarity.  
- Nearby Alerts – Automatically display incidents within a user’s vicinity.  
- Modular Backend – Handles storage, validation, and data flow between users.

---

## Tech Stack
Frontend: HTML, CSS, JavaScript, Leaflet.js  
Backend: Node.js, Express.js  
Database: MongoDB Atlas  
Hosting: Compatible with local or cloud-based deployment.

## Setup Instructions

### 1. Clone the Repository
```bash
git clone https://github.com/Mithubala123/disaster-backend.git
cd disaster-backend
````

### 2. Install Dependencies

```bash
npm install
```

### 3. Run the Server

```bash
node server.js
```

### 4. Launch the Frontend

Open `index.html` in your browser or start your frontend server if configured.
